mandatehub

Provable autonomous machine-to-machine payment.

A small, self-contained verification core. It does not execute payments on chain — it makes an autonomous agent's spending provably bounded, best-executed, and honestly settled, as deterministic, offline-verifiable artifacts. Standard library only, zero runtime dependencies.

🟢 Live service — mandatehub.obolpay.xyz

A running x402 resource server with a mandate gate: budget-capped, replay-proof, proof-carrying payments, settling real USDC on Base via Coinbase CDP. Pay it with any x402 client:

pip install 'mandatehub[evm]'
export MANDATEHUB_AGENT_PRIVATE_KEY=0x...   # a Base-funded key
python -m examples.x402_pay https://mandatehub.obolpay.xyz/quote
# → 200 + data + on-chain settlement tx + ProofOfMandate

Endpoints: / · /healthz · /metrics · /quote (402 → pay). Self-funded pilot; mainnet is the owner's call, not an audited guarantee.

Intent / account abstraction (④)

A mandate is a pre-funded, budget-bounded authorization. An agent settles intents within it, and a ProofOfMandate lets anyone verify offline that the budget was never exceeded.

Best execution / recapture (③)

A solver auction fills at the best disclosed cost; the price-improvement surplus is split integer-exactly, with a best-execution proof and a surplus-recapture proof.

Design identity

  • Deterministic + offline-verifiable — explicit time, never the wall clock; same inputs → byte-identical hashes.
  • Append-only double-entry — integer minor units, every transaction balances to zero per currency.
  • No on-chain execution, no HTTP, no keys in core — it proves the accounting; the chain pieces live around it.
